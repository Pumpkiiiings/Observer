# Animaciones

Observer tiene soporte incorporado para disparar animaciones de Blockbench en los modelos de los jugadores.

<Callout type="info">
**Dependencia Requerida**

El soporte de animaciones depende del increíble mod [Player Animator](https://modrinth.com/mod/player-animation-library) de KosmX. 
¡Asegúrate de que tus jugadores tengan este mod instalado en su cliente junto con Observer para que las animaciones funcionen!
</Callout>

---

### `PaperObserverAnimationAPI`
> Dispara animaciones de Blockbench en los modelos de los jugadores.

`com.observer.paper.api.PaperObserverAnimationAPI`

**Firma**
```java
public class PaperObserverAnimationAPI
```

**Métodos**

#### `playAnimation`
```java
public static void playAnimation(Player viewer, Player target, String animationName)
```
Reproduce una animación en el jugador `target`, siendo visible para el `viewer`.

**Ejemplo**
```java
import com.observer.paper.api.PaperObserverAnimationAPI;
import com.observer.paper.api.events.ObserverPlayerSprintEvent;
import org.bukkit.event.EventHandler;
import org.bukkit.event.Listener;

public class AnimationListener implements Listener {

    @EventHandler
    public void onSprint(ObserverPlayerSprintEvent event) {
        if (event.isSprinting()) {
            // Reproduce la animación "naruto_run" en el jugador que empezó a correr,
            // y muéstrasela a todos los jugadores en su mundo.
            for (Player espectador : event.getPlayer().getWorld().getPlayers()) {
                PaperObserverAnimationAPI.playAnimation(espectador, event.getPlayer(), "naruto_run");
            }
        }
    }

    @EventHandler
    public void onWalk(ObserverPlayerWalkEvent event) {
        if (event.isWalking()) {
            PaperObserverAnimationAPI.playAnimation(event.getPlayer(), "custom_walk");
        } else {
            PaperObserverAnimationAPI.stopAnimation(event.getPlayer());
        }
    }

    @EventHandler
    public void onJump(ObserverPlayerJumpEvent event) {
        PaperObserverAnimationAPI.playAnimation(event.getPlayer(), "backflip");
    }

    @EventHandler
    public void onLeftClick(ObserverPlayerLeftClickEvent event) {
        PaperObserverAnimationAPI.playAnimation(event.getPlayer(), "sword_slash");
    }

    @EventHandler
    public void onIdle(ObserverPlayerIdleEvent event) {
        if (event.isIdle()) {
            event.getPlayer().sendMessage("¡Has entrado en estado inactivo!");
            // El plugin base reproducirá la animación configurada automáticamente.
        } else {
            event.getPlayer().sendMessage("¡Te has vuelto a mover!");
        }
    }
}
```

### Configuración Automática (Idle)

Observer incluye un sistema integrado para reproducir animaciones automáticamente cuando el jugador está inactivo (Idle). Esto se configura en `plugins/Observer/config.yml`.

```yaml
animations:
  idle:
    enabled: true
    animation-name: "idle" # El nombre de tu animación en Blockbench
    threshold-ms: 1000     # Tiempo en milisegundos que el jugador debe estar quieto
```

Cuando un jugador se detiene durante el tiempo especificado (`threshold-ms`), el plugin dispara automáticamente la animación definida. Cuando el jugador vuelve a moverse, el plugin envía un paquete para detenerla.
